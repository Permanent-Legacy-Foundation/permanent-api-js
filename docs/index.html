Introduction

This guide is for developers who want to use the permanent.org REST API for Digital Preservation as a Service.

If would like to learn more about permanent.org, go here. To learn more about digital archives, go here.

About this API reference

This API reference is organized by resource type. Each resource type has one or more data representations and one or more methods.

Getting Started

To get up to speed quickly, it's best to login to permanent.org and use the interface to become familiar with archives and all of the exciting features available.

Summary of how permanent.org works

A person creates an account with an email address and password. With an account, you can create archives for a person, family, or organization. Files can then be uploaded to folders in an archive.

Summary of how the API works

The REST API currently supports basic features for creating archives, uploading and fetching records, creating and fetching folders, as well as some basic account features.

Let's begin
  1. Create an account on permanent.org.
  2. Request an API key from permanent.org by contacting support@permanent.org
  3. Use the API key supplied by permanent.org in your API calls.
About your API key

Keep your API key private and secure.
API keys are for making server side requests only. It should not be used in client side code where it would be publicly exposed.

How to use the REST API

Sending data always requires the HTTP Post method type sent using HTTPS. The HTTP header Content-Type should be 'application/x-www-form-urlencoded' for most calls. For sending files, use the Content-Type = 'multipart/form-data'. All data sent in API requests must be JSON formatted strings set to a body variable called 'form';

The Request and Response Model

...about the model

The structure of the request body.

                      
                        body 
                          form = 
                              {
                                apiKey:'string',
                                data:{}            
                              }

A simple example of an application/x-www-form-urlencoded request. 

                      
                        POST /archive/add HTTP/1.1
                        Host: api.permanent.org
                        Content-Type: application/x-www-form-urlencoded
                        Content-Length: 27
                        
                        form="{"apiKey":"your-apikey","data":{"fullname":"a-value","type":"person"}}"

A simple example of an multipart/form-data request. 

                        

                        POST /record/add HTTP/1.1 
                        Host: api.permanent.org
                        Content-Type: multipart/form-data;boundary="boundary" 
                        
                        --boundary 
                        Content-Disposition: form-data; name="form" 
                        
                        "{"apiKey":"your-apikey","data":{"archiveNumber":"0000-0000","record_name":"my-file"},
                        "file":{"name":"permanent-logo.png","contentType":"the-mime-type"}}" 

                        --boundary 
                        Content-Disposition: form-data; name="permanent-logo"; filename="permanent-logo.png" 
                        
                        the-file-byte-array
                        --boundary--

This guide uses examples written in javascript for Node JS server side API calls. However, you can create your API wrapper in the programming language of your choice.

An Archive is the primary resource for organizing content for a person, family, or organization. An account can create as many Archives as they wish.



Methods Route Description

add - /archive/add

Add a new archive to an account.

Request
Property Value Description
fullname string Required. The name of the archive.
type string Required. Acceptible values are person,family, and organization.

Response
Property Value Description
data.archiveNumber string The unique archive number.
success boolean true or false
error string An error code.
message string Any helpful messages regarding the request.
httpcode string The HTTP response code.

Example
                 
                    var req = {
                      apiKey: config.apikey,
                      data: {
                        fullname: "My archive name",
                        type: 'person'
                      }
                    };
                
                    send_HTTP_Post('/archive/add', req).then(function (response) {
                      assert.isTrue(response.success);
                      assert.isEmpty(response.error);
                      assert.isEmpty(response.message);
                      assert.isTrue(response.httpcode == 200);
                      assert.isString(response.data.archiveNumber, 'new archive number is: ' + response.data.archiveNumber);
                      done();
                    });
get - /archive/get

Get an archive.

Request
Property Value Description
archiveNumber string Required. The archive number for the archive.

Response
Property Value Description
data.archiveNumber string The unique archive number.
data.fullname string The name of the archive.
data.thumbnails object A list of urls for thumbnail images of various sizes.
success boolean true or false
error string An error code.
message string Any helpful messages regarding the request.
httpcode string The HTTP response code.

Example
                   
                      var req = {
                        apiKey: 'apikey',
                        data: {
                          archiveNumber: '0000-0000'
                        }
                      };
                  
                      send_HTTP_Post('/archive/get', req).then(function (response) {
                        assert.isTrue(response.success);
                        assert.isEmpty(response.error);
                        assert.isEmpty(response.message);
                        assert.isTrue(response.httpcode == 200);
                        assert.isObject(response.data.archive);
                        assert.isNotEmpty(response.data.archive.archiveNumber);
                        assert.isNotEmpty(response.data.archive.fullname);
                        assert.isNotEmpty(response.data.archive.thumbnails.thumbURL1000);
                        assert.isNotEmpty(response.data.archive.thumbnails.thumbURL2000);
                        assert.isNotEmpty(response.data.archive.thumbnails.thumbURL200);
                        assert.isNotEmpty(response.data.archive.thumbnails.thumbURL500);
                      
                      });

add - /record/add

Add a record to an archive.

Request
Property Value Description
archive object
- paNumber string
Required
The archive's PA Number
record object
- name string
Not Required
A user entered name of the file.
file object
name Type: string; Not Required
The upload file name with extension.

The contents of the file must be sent along with the post as multipart/form-data.



Response
Property Value Description
data.archiveNumber string The unique archive number.
success boolean true or false
error string An error code.
message string Any helpful messages regarding the request.
httpcode string The HTTP response code.



Example
                     
                      var req = {
                        apiKey: config.apikey,
                        data: {
                          archive: {
                            paNumber: 'the_archive_paNumber'
                          },
                          record: {
                            name: 'permanent-logo',
                            description: 'a description of the record',
                            location: {
                              displayName: 'home',
                              streetNumber: '4611',
                              streetName: 'Bee Caves Road',
                              postalCode: '78746',
                              locality: 'West Lake Hills',
                              adminOneName: 'Texas',
                              adminOneCode: 'TX',
                              adminTwoName: 'Travis County',
                              adminTwoCode: 'Travis County',
                              country: 'United States',
                              countryCode: 'US',
                              latitude: '30.286603',
                              longitude: '-97.814744'
                            }
                          }
                        },
                        file: {
                          name: "permanent-logo.png"
                        }
                      };
                    
                        //here you need to send as Content-Type: multipart/form-data;
                        //be sure to include the file in the post body

                        var filepath ='path_to_my_file.txt';

                        send_HTTP_Multipart_form_Post('/record/add',filepath, req).then(function (res) {
                          assert.isTrue(res.success);
                          assert.isTrue(res.httpcode==200);
                          assert.isEmpty(res.error);
                          assert.isEmpty(res.message);
                          assert.isObject(res.data);
                          assert.isObject(res.data.record);
                          assert.isNotEmpty(res.data.record.archiveNumber);
                          assert.isString(res.data.record.paNumber, 'new archive number is: ' + res.data.record.paNumber);
                        
                        });
